Component Lifecycle Hooks

üîÑ Lifecycle Execution Order

Definition:

Angular calls lifecycle hooks in a specific order during component creation, updates, and destruction. Understanding this order is crucial for proper component behavior.

1

constructor()

TypeScript class constructor. Initialize basic properties, inject dependencies.

Before Angular initialization
2

ngOnChanges()

Called when @Input properties change. Receives SimpleChanges object.

Before ngOnInit and whenever @Input changes
3

ngOnInit()

Initialize component after Angular sets input properties. Perfect for setup logic.

Once after first ngOnChanges
4

ngDoCheck()

Custom change detection. Called during every change detection cycle.

After ngOnChanges and ngOnInit, then every cycle
5

ngAfterContentInit()

Called after Angular projects external content into component's view.

Once after first ngDoCheck
6

ngAfterContentChecked()

Called after Angular checks projected content.

After ngAfterContentInit and every ngDoCheck
7

ngAfterViewInit()

Called after Angular initializes component's views and child views.

Once after first ngAfterContentChecked
8

ngAfterViewChecked()

Called after Angular checks component's views and child views.

After ngAfterViewInit and every ngAfterContentChecked
9

ngOnDestroy()

Cleanup before Angular destroys the component. Unsubscribe, detach event handlers.

Just before component destruction

üìö Detailed Hook Explanations

üèóÔ∏è Initialization Hooks

constructor()

Purpose: Basic initialization, dependency injection

When to use: Set default values, inject services

Avoid: DOM manipulation, heavy operations

ngOnInit()

Purpose: Component initialization after input binding

When to use: API calls, complex initialization, setup subscriptions

Best practice: Most initialization logic goes here

üîÑ Change Detection Hooks

ngOnChanges(changes: SimpleChanges)

Purpose: Respond to @Input property changes

When to use: React to input changes, validate inputs

Parameters: SimpleChanges object with previous/current values

ngDoCheck()

Purpose: Custom change detection logic

When to use: Detect changes Angular can't detect automatically

Warning: Called frequently, keep logic lightweight

üì¶ Content Hooks

ngAfterContentInit()

Purpose: Access projected content (ng-content)

When to use: Initialize logic that depends on projected content

Safe for: Accessing @ContentChild elements

ngAfterContentChecked()

Purpose: Respond to projected content changes

When to use: React to content updates

Warning: Called frequently during change detection

üëÅÔ∏è View Hooks

ngAfterViewInit()

Purpose: Access ViewChild elements, initialize third-party libraries

When to use: DOM manipulation, ViewChild access

Safe for: Accessing child components and DOM elements

ngAfterViewChecked()

Purpose: Respond to view changes

When to use: React to view updates

Warning: Called frequently during change detection

üßπ Cleanup Hook

ngOnDestroy()

Purpose: Cleanup before component destruction

When to use: Unsubscribe observables, clear timers, remove event listeners

Critical for: Preventing memory leaks
