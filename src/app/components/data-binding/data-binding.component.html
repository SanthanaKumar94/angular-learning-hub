Data Binding in Angular

1. Interpolation {{ '{' }}{{ '{' }} {{ '}' }}{{ '}' }}

Definition:

Interpolation allows you to embed expressions into marked up text. It's a one-way data binding technique that displays component data in the template using double curly braces.

Syntax: {{ '{' }}{{ '{' }} expression {{ '}' }}{{ '}' }}

Data Flow: Component → Template (One-way)

Live Examples:

Welcome, {{ userName() }}!

Today is {{ currentDate() }}

2 + 3 = {{ 2 + 3 }}

Message: {{ getMessage() }}

{{ interpolationCode }}

2. Property Binding [property]

Definition:

Property binding sets a property of a target element or directive. It binds component properties to DOM element properties, not HTML attributes.

Syntax: [property]="expression"

Data Flow: Component → Template (One-way)

Use Case: Setting element properties like src, disabled, value, etc.

Live Examples:







{{ propertyBindingCode }}

3. Event Binding (event)

Definition:

Event binding listens for and responds to user actions such as keystrokes, mouse movements, clicks, and touches. It allows data to flow from the template to the component.

Syntax: (event)="method()"

Data Flow: Template → Component (One-way)

Use Case: Handling user interactions like clicks, input changes, mouse events.

Live Examples:

Button clicked {{ clickCount() }} times

You typed: {{ typedText() }}

{{ eventBindingCode }}

4. Two-Way Binding [(ngModel)]

Definition:

Two-way binding combines property binding and event binding in a single notation. It allows data to flow both ways: from component to template and from template to component.

Syntax: [(ngModel)]="property"

Data Flow: Component ↔ Template (Bidirectional)

Requirements: Import FormsModule in component

Use Case: Form controls where user input should update component data immediately.

Live Examples:

Hello, {{ formData.name }}!

You are {{ formData.age }} years old

Your message: {{ formData.message }}

{{ twoWayBindingCode }}

5. Attribute Binding [attr.attribute]

Definition:

Attribute binding sets the value of HTML attributes directly. Use it when there's no corresponding DOM property or when you need to set ARIA attributes, data attributes, or SVG attributes.

Syntax: [attr.attribute-name]="expression"

Data Flow: Component → Template (One-way)

Use Case: Setting HTML attributes like aria-label, data-*, colspan, etc.

Live Examples:

6. Class Binding [class]

Definition:

Class binding allows you to add and remove CSS classes dynamically. You can bind to a single class conditionally or set multiple classes at once.

Syntax: [class.class-name]="condition" or [class]="expression"

Data Flow: Component → Template (One-way)

Use Case: Dynamic styling, conditional CSS classes, theme switching.

Live Examples:

Dynamic class binding
Conditional classes

7. Style Binding [style]

Definition:

Style binding allows you to set inline styles dynamically. You can bind to individual style properties or set multiple styles at once using a style object.

Syntax: [style.property]="value" or [style]="styleObject"

Data Flow: Component → Template (One-way)

Use Case: Dynamic styling, animations, responsive design, theme customization.
